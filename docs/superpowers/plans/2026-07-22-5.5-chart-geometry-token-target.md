# Script-readable token target Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Give Arena a second build target that emits flagged DTCG tokens as JavaScript numbers into each framework layer, and delete the ten hand-declared geometry constants those tokens replace.

**Architecture:** `tokens/src/` stays the only place a value is authored. A token opts into the JS target with `$extensions["com.dravensoft.arena"].script: true` and then emits **twice** — the CSS custom property it already would have, and a named export in a generated module. The module is emitted **per framework layer** in that layer's own language (`frameworks/react/tokens.generated.js`, `frameworks/angular/tokens.generated.ts`), so a component's import never crosses the `tokens/` ↔ `frameworks/` boundary. A parity gate asserts the modules match both the source and the CSS.

**Tech Stack:** Bun (build + test), Style Dictionary v4 (loader/resolver only — never a value transformer or formatter), `node:test`/`node:assert`, plain ES modules.

## Source spec

`docs/superpowers/specs/2026-07-19-5.5-chart-geometry-token-target-design.md`

**Read its *What 2026-07-22 settled* section before its §5.** §5 is marked SUPERSEDED: it describes a single module under `tokens/`, which was rejected. This plan implements the per-layer design.

## Global Constraints

- **English only** — all code, comments, docs and copy.
- **No emoji**, in product or docs.
- **`tokens/src/` is authored; `tokens/*.css` is generated.** Never hand-edit `palette.css`, `typography.css`, `spacing.css`, `effects.css`. Same now applies to `frameworks/*/tokens.generated.*`.
- **Style Dictionary is a loader only.** Values are rendered by `scripts/lib/serialize-token.mjs` and this plan's new `scripts/lib/serialize-script.mjs`. Never enable an SD value transform or `format`.
- **`load()` in `build-tokens.mjs` loads one source file per call**, so DTCG aliases (`"{sp.2}"`) do **not** resolve across files. Author values directly; record equivalences in `$description`.
- **DTCG 2025.10 strictness:** dimensions and durations are `{value, unit}` objects; `number` is bare. `$extensions` keys must be reverse-DNS (`check-dtcg.mjs:91`), so `com.dravensoft.arena` needs no gate change.
- **Commit generated output.** The plugin is served from the release tag and the copy-in kit reads files directly.
- **`bun run check` is a completion gate**, run once at the end (Task 10) — not per commit. Individual gates are cheap and run per task.
- **CHANGELOG entries go under `## [Unreleased]`**, never under the last version.

## Design decisions this plan settles

The spec left six open questions. This plan answers five and implements a narrowed sixth. Each is recorded in the task that carries it.

| # | Question | Decision |
|---|---|---|
| 3 | Bar radius: name `4` or snap to `--r-sm` (6)? | **Name it.** `chart.bar-radius` = 4px. Snapping is a visible 2px design change and the spec says it must be taken deliberately, not smuggled into a refactor. Task 1. |
| 4 | `PAD.b`/`PAD.l`: four tokens or one gutter? | **Four tokens.** 28 ≠ 44, so one gutter token cannot serve both. `$description` records that they are label gutters sized from the type scale, so a future type change knows to revisit. Task 1. |
| 5 | Are the ratios tokens? | **No.** `0.62`, `0.34` and `ticks()`' default `4` stay in code. Precedent is `TYPE-MAP.md`'s Avatar row, which documents `× 0.4` and `× 0.28` as derivations rather than tokens. Consistency matters more than the individual call. Task 4. |
| 8 | `CAT_SLOTS`: authored or derived? | **Derived.** The build counts the `cat-*` tokens in `palette.dark.json` and emits `catSlots`. Authoring it would re-create the second-place-to-update problem it is being removed for. Task 6. |
| 9 | Onboarding's `W`/`EDGE` | **Both, plus the `12` beside them.** `EDGE` is `--sp-4` and `12` is `--sp-3` — existing tokens, flagged rather than re-authored. `W` = 320 has no token and gains `onboarding.width`, which also collapses the `calc(var(--sp-1) * 80)` duplicate the component renders. Task 8. |
| 6 | What shape does the "catch the next `CHART_HEIGHT`" gate take? | **A duplication gate, which is narrower than the spec hoped.** See Task 9 for what it does and does not catch. |

A token carrying `$extensions["com.dravensoft.arena"].script: true` is emitted

- [ ] **Step 3: Write the serializer**

Create `scripts/lib/serialize-script.mjs`:

```js
/* Renders a script-readable DTCG token to the bare JavaScript number the
 * generated per-layer modules export.
 *
 * Sibling of serialize-token.mjs and under the same contract: Style Dictionary
 * never transforms these values, Arena renders them. Where serialize-token.mjs
 * produces the CSS string ("280px"), this produces the number JS does
 * arithmetic on (280). The parity gate (scripts/check-script-tokens.mjs) is
 * what holds the two in step.
 *
 * Only dimension, duration and number are script-readable, because they are the
 * only DTCG types whose value IS a number. A color or a shadow has no bare
 * numeric form, and flagging one is a mistake this refuses rather than guesses
 * at. */

/** Types whose $value reduces to a single number. */
const NUMERIC = new Set(['dimension', 'duration', 'number']);

/** @param {{ $type?: string, $value: unknown }} token @returns {number} */
export function serializeScript(token) {
  if (!NUMERIC.has(token.$type)) {
    throw new Error(`serializeScript: $type "${token.$type}" is not script-readable`);
  }
  if (token.$type === 'number') return token.$value;
  return token.$value.value;
}

/** "chart-pad-left" -> "chartPadLeft". The build's name/kebab transform has
 *  already run, so this only has to camelCase what it produced. */
export function scriptName(kebab) {
  return kebab.replace(/-([a-z0-9])/g, (_, c) => c.toUpperCase());
}
```

After the `FILES` array, add:

```js
const EXT = 'com.dravensoft.arena';

/** The generated JS modules, one per framework layer. Emission is per-layer
 *  rather than once into tokens/ so that a component's import never crosses the
 *  tokens/ <-> frameworks/ boundary: the import is intra-layer, the same shape
 *  as the ../charts/chart-internals.js that Calendar already carries, and the
 *  copy-in kit gains one entry in README.md's dependency list rather than a new
 *  concept. Both files carry identical bodies; only the extension differs,
 *  because React does not compile TypeScript and Angular needs a .ts to
 *  typecheck the import. */
export const SCRIPT_TARGETS = [
  'frameworks/react/tokens.generated.js',
  'frameworks/angular/tokens.generated.ts',
];

const SCRIPT_HEADER =
  '/* GENERATED by scripts/build-tokens.mjs — edit tokens/src/, not this file.\n'
  + ' *\n'
  + ' * Every export here is a token flagged script-readable in tokens/src/ and\n'
  + ' * emitted as a bare number because JS arithmetic consumes it. Each one also\n'
  + ' * ships as a CSS custom property; scripts/check-script-tokens.mjs asserts the\n'
  + ' * two carry the same value. A value bound at import time cannot re-theme or\n'
  + ' * re-densify — that is the accepted price of reading it in JS at all. */';

/** True when a resolved token opted into the JS target. */
const isScript = (token) => token.$extensions?.[EXT]?.script === true;

/** Every source file the CSS build reads, de-duplicated and in declaration
 *  order. Scanning FILES rather than a second list is what keeps the two
 *  targets from disagreeing about which sources exist. */
function scriptSources() {
  return [...new Set(FILES.flatMap((f) => f.blocks.map((b) => b.source)))];
}

/** @returns {Promise<Array<{name: string, value: number}>>} flagged tokens, in source order */
async function collectScriptTokens() {
  const out = [];
  for (const source of scriptSources()) {
    for (const item of walk(await load(source))) {
      if (item.group || !isScript(item.token)) continue;
      out.push({
        name: scriptName(item.token.name),
        value: serializeScript(item.token),
        $description: item.token.$description,
      });
    }
  }
  return out;
}

/** @returns {Promise<Map<string,string>>} repo-relative path -> module source */
export async function buildScriptModules() {
  const tokens = await collectScriptTokens();
  const lines = [SCRIPT_HEADER, ''];
  for (const t of tokens) {
    if (t.$description) lines.push(...comment(t.$description).split('\n').map((l) => l.replace(/^ {2}/, '')));
    lines.push(`export const ${t.name} = ${t.value};`);
  }
  const body = `${lines.join('\n')}\n`;
  return new Map(SCRIPT_TARGETS.map((path) => [path, body]));
}
```

Then extend `main()` so the modules are written alongside the CSS:

```js
async function main() {
  const built = await buildAll();
  for (const [name, css] of built) {
    writeFileSync(join(root, 'tokens', name), css);
    console.log(`build-tokens: wrote tokens/${name}`);
  }
  const modules = await buildScriptModules();
  for (const [path, source] of modules) {
    writeFileSync(join(root, path), source);
    console.log(`build-tokens: wrote ${path}`);
  }
}
```

Three assertions, per the spec's §3 as revised on 2026-07-22:

1. **Drift** — the committed modules are what the source generates.
2. **Parity** — every export has a CSS counterpart carrying the same number.
3. **No orphan flags** — every flagged token is imported by **at least one** layer. "At least one" is deliberate: `calendarHourH` is real and React-only, because React has a `Calendar` and Angular does not.

Create `scripts/check-script-tokens.mjs`:

```js
/* Asserts the committed frameworks/*/tokens.generated.* are what tokens/src/
 * generates, that each export agrees with its CSS counterpart, and that no
 * token is flagged script-readable without anything importing it.
 *
 * The generated modules are committed (the plugin is served from the release
 * tag and the copy-in kit reads them directly), so a stale committed file is a
 * silent failure. This is the guard, and it is the JS-side twin of
 * check-tokens-generated.mjs.
 *
 *   bun scripts/check-script-tokens.mjs   -> exit 0 if in sync, 1 on drift
 */
import { readFileSync, readdirSync, statSync } from 'node:fs';
import { fileURLToPath } from 'node:url';
import { dirname, join, extname } from 'node:path';
import { buildScriptModules, SCRIPT_TARGETS } from './build-tokens.mjs';
import { parseDecls } from './lib/css-decls.mjs';

const root = join(dirname(fileURLToPath(import.meta.url)), '..');

/** The CSS half of a script-readable token is always a bare number with at most
 *  a px/ms unit — that is what serialize-token.mjs renders for the three
 *  flaggable types. Anything else means the token is not script-readable and
 *  the flag is wrong. */
export function cssCounterpart(value) {
  const m = /^(-?\d+(?:\.\d+)?)(px|ms)?$/.exec(value.trim());
  return m ? Number(m[1]) : null;
}

/** Every name imported from a `tokens.generated.*` module in one source file. */
export function importedNames(source) {
  const names = new Set();
  const re = /import\s*\{([^}]*)\}\s*from\s*['"][^'"]*tokens\.generated(?:\.js|\.ts)?['"]/g;
  for (const m of source.matchAll(re)) {
    for (const raw of m[1].split(',')) {
      const name = raw.trim().split(/\s+as\s+/)[0].trim();
      if (name) names.add(name);
    }
  }
  return names;
}

const SCAN_EXT = new Set(['.js', '.jsx', '.ts', '.tsx']);

function* sourceFiles(dir) {
  for (const entry of readdirSync(dir)) {
    if (entry === 'node_modules' || entry === 'vendor') continue;
    const path = join(dir, entry);
    if (statSync(path).isDirectory()) { yield* sourceFiles(path); continue; }
    if (!SCAN_EXT.has(extname(entry))) continue;
    if (entry.startsWith('tokens.generated.')) continue;
    yield path;
  }
}

const problems = [];

/* 1. Drift. */
const built = await buildScriptModules();
for (const [path, expected] of built) {
  let actual;
  try {
    actual = readFileSync(join(root, path), 'utf8');
  } catch {
    problems.push(`${path}: missing — run bun run build:tokens`);
    continue;
  }
  if (actual !== expected) problems.push(`${path}: stale — run bun run build:tokens`);
}

/* 2. Parity against the CSS. */
const cssValues = new Map();
for (const file of readdirSync(join(root, 'tokens'))) {
  if (extname(file) !== '.css') continue;
  for (const [, decls] of parseDecls(readFileSync(join(root, 'tokens', file), 'utf8'))) {
    for (const [prop, value] of decls) if (!cssValues.has(prop)) cssValues.set(prop, value);
  }
}

const exported = new Map();
for (const m of built.values().next().value.matchAll(/^export const (\w+) = (-?[\d.]+);$/gm)) {
  exported.set(m[1], Number(m[2]));
}

/** "chartPadLeft" -> "chart-pad-left", to find the custom property again. */
const kebab = (camel) => camel.replace(/([a-z0-9])([A-Z])/g, '$1-$2').toLowerCase();

for (const [name, value] of exported) {
  const prop = kebab(name);
  if (!cssValues.has(prop)) {
    problems.push(`${name}: exported to JS but --${prop} is not in any tokens/*.css`);
    continue;
  }
  const css = cssCounterpart(cssValues.get(prop));
  if (css === null) {
    problems.push(`${name}: --${prop} is "${cssValues.get(prop)}", which is not a bare number — the script flag is wrong`);
  } else if (css !== value) {
    problems.push(`${name}: JS has ${value}, --${prop} has ${css}`);
  }
}

/* 3. No orphan flags. */
const imported = new Set();
for (const path of sourceFiles(join(root, 'frameworks'))) {
  for (const name of importedNames(readFileSync(path, 'utf8'))) imported.add(name);
}
for (const name of exported.keys()) {
  if (!imported.has(name)) {
    problems.push(`${name}: flagged script-readable but no framework layer imports it — remove the flag or use the token`);
  }
}

if (problems.length) {
  console.error(`check-script-tokens: ${problems.length} problem(s)\n`);
  for (const p of problems) console.error(`  ${p}`);
  process.exit(1);
}
console.log(`check-script-tokens: ${exported.size} script-readable token(s) in sync across ${SCRIPT_TARGETS.length} layer(s)`);
```

```js
/* Fails when the same named numeric constant is declared in BOTH framework
 * layers, which is how chart geometry drifted before the script-readable token
 * target existed: CAT_SLOTS, CHART_HEIGHT and PAD were declared identically in
 * frameworks/react/components/charts/chart-internals.js and
 * frameworks/angular/primitives/chart-internals.ts, and W and EDGE in the two
 * Onboarding implementations. All five would have failed here the day the
 * second one was written.
 *
 * WHAT THIS DOES NOT CATCH, stated so nobody reads it as more than it is: a
 * design value declared in ONE layer only. Deciding whether a bare number in a
 * JS object is a design value needs judgement no scanner has -- a number in an
 * object is not a dimension until something uses it as one, which is exactly
 * why check-dimension-literals.mjs cannot reach these. This gate takes the
 * decidable half: cross-layer duplication, no judgement call, near-zero false
 * positives.
 *
 *   bun scripts/check-duplicate-constants.mjs   -> exit 0 if clean, 1 on duplication
 */
import { readFileSync, readdirSync, statSync } from 'node:fs';
import { fileURLToPath } from 'node:url';
import { dirname, join, extname, relative } from 'node:path';

const root = join(dirname(fileURLToPath(import.meta.url)), '..');

/* A constant that is legitimately the same in both layers because it is the
 * same external fact, not an Arena design decision. Each entry carries a
 * reason, and a stale entry -- one naming a constant no longer duplicated --
 * fails this gate, the same invariant check-dimension-literals.mjs's EXEMPT
 * holds. */
const EXEMPT = new Map([
  // (empty today; add with a reason when a genuine shared constant appears)
]);

/** Module-level `const NAME = <number>` and `const NAME = { k: <number>, ... }`.
 *  Deliberately only module level: a constant inside a function body is local
 *  reasoning, not a shared value, and Onboarding's own W/EDGE lived in a
 *  function before this plan moved them out. */
export function numericConstants(source) {
  const found = new Map();
  const re = /^(?:export\s+)?const\s+([A-Za-z_$][\w$]*)\s*=\s*([^;]+);/gm;
  for (const m of source.matchAll(re)) {
    const raw = m[2].replace(/\s*as\s+const\s*$/, '').trim();
    if (/^-?\d+(\.\d+)?$/.test(raw)) { found.set(m[1], raw); continue; }
    if (/^\{[^{}]*\}$/.test(raw)) {
      const body = raw.slice(1, -1).trim();
      if (body && /^([\w$]+\s*:\s*-?\d+(\.\d+)?\s*,?\s*)+$/.test(body)) {
        found.set(m[1], `{${body.replace(/\s+/g, '').replace(/,$/, '')}}`);
      }
    }
  }
  return found;
}

const SCAN_EXT = new Set(['.js', '.jsx', '.ts', '.tsx']);

function* sourceFiles(dir) {
  for (const entry of readdirSync(dir)) {
    if (entry === 'node_modules' || entry === 'vendor') continue;
    const path = join(dir, entry);
    if (statSync(path).isDirectory()) { yield* sourceFiles(path); continue; }
    if (!SCAN_EXT.has(extname(entry))) continue;
    if (entry.startsWith('tokens.generated.')) continue;
    // A compiled .js sibling restates its .jsx source; scanning both would
    // report every constant as duplicated with itself.
    if (extname(entry) === '.js' && readdirSync(dir).includes(`${entry.slice(0, -3)}.jsx`)) continue;
    yield path;
  }
}

/** name -> layer -> [{ file, value }] */
const byName = new Map();
for (const layer of ['react', 'angular']) {
  const dir = join(root, 'frameworks', layer);
  for (const path of sourceFiles(dir)) {
    for (const [name, value] of numericConstants(readFileSync(path, 'utf8'))) {
      if (!byName.has(name)) byName.set(name, new Map());
      const layers = byName.get(name);
      if (!layers.has(layer)) layers.set(layer, []);
      layers.get(layer).push({ file: relative(root, path), value });
    }
  }
}

const problems = [];
const hit = new Set();

for (const [name, layers] of byName) {
  if (layers.size < 2) continue;
  hit.add(name);
  if (EXEMPT.has(name)) continue;
  const where = [...layers].map(([layer, decls]) =>
    decls.map((d) => `${d.file} = ${d.value}`).join(', ')).join('  and  ');
  problems.push(`${name}: declared in both layers — ${where}\n    Author it in tokens/src/ with the script flag instead.`);
}

for (const name of EXEMPT.keys()) {
  if (!hit.has(name)) {
    problems.push(`EXEMPT entry "${name}" is stale — it is no longer declared in both layers. Remove it.`);
  }
}

if (problems.length) {
  console.error(`check-duplicate-constants: ${problems.length} problem(s)\n`);
  for (const p of problems) console.error(`  ${p}`);
  process.exit(1);
}
console.log('check-duplicate-constants: no numeric constant is declared in both framework layers');
```
