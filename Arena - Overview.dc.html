Components
Buttons and actions
Danger convention
Destructive actions are not filled like the primary action. They use a transparent background with the border and content (text and icons) in the semantic token --error. This way danger reads through color, not visual weight, and never competes with the crimson primary button.
primary · filled --primary
danger · outline --error
exception · filled only when confirming the irreversible
  • Do reserve the solid fill in --error only for the last irreversible step inside a confirmation dialog.
  • Don't use filled danger buttons as triggers in lists, cards or toolbars.
  • Do apply the same rule to danger icon buttons: .iconbtn.danger is also transparent with an --error border.
